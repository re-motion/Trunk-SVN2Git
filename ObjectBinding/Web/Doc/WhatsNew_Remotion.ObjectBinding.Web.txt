- Umstellung auf re:motion
- Umstellung von NullableValueTypes auf .NET-Nullables (z.B. NaInt32 => int?)

  Besonders wichtig: In <listControlStyle>-Tags 'radiobuttonlistcellspacing=""' und 'radiobuttonlistcellpadding=""' per Suchen/Ersetzen entfernen.

1.4.3
======
- Namespace geändert:
  * Rubicon.ObjectBinding.Web.Controls zu Rubicon.ObjectBinding.Web.UI.Controls
  * Rubicon.ObjectBinding.Web.Design zu Rubicon.ObjectBinding.Web.UI.Design
  * Best Practice für Migration der Projekte:
    1. Alle betroffenen Projekte auschecken
    2. Search & Replace auf alle Files durchführen um den alten Namespace durch den neuen zu ersetzen.
       Funktioniert sowaohl für aspx/ascx- als auch für Code-Behind- und resx-Files.
     
- BusinessObjectReferenceDataSourceControl
  * Das Verhalten von LoadValues und SaveValues überarbeitet: Jetzt das BusinessObject selbst nur gespeichert,
    wenn es nicht aus der referenzierten DataSource geladen wurde. 
    (zB bei On-The-Fly Generierung, bei direktem Setzen,...)
  * Für gebundene ReferenceDataSourcen ergibt sich damit folgende Änderung (bzw Bugfix):
    Wurde ein Object On-The-Fly generiert, so wurde es bisher nur in die refernzierte DataSource zurückgeschrieben,
    wenn die ObjectClass RequiresWriteBack gesetzt hatte. 
  
- BOCs mit Editmode:
  * DirtyState überarbeitet:
    Bisheriges Verhalten: IsDirty war immer true, außer es wurde LoadValues aufgerufen.
    Neues Verhalten: 
      Initialisiert: IsDirty = false
      set Value: IsDirty = true
      LoadValue/LoadUnboundValue: IsDirty = false, wenn der Wert im Control dem zugewiesenen Wert entspricht.
      Durch Interaktion: IsDirty = true wenn sich durch UserInput (oder bei der BocList zB AddRow) der Wert von Value 
        ändert.

      IsDirty wird gelöscht, sobald SaveValue aufgerufen wird und Value wieder in die DataSource zurückgeschrieben
      wird.
      Für die BocList gilt das auch bei SaveValue(interim=true). Im Regelfall wird SaveValue(interim=true) im 
      Page.Unload aufgerufen und damit kann der neue IsDirty Zustand nicht mehr im ViewState perisistiert werden.
      
      Überarbeitet in 1.4.34: Nur bei interim=false wird das DirtyFlag gelöscht.
    
- BOCs:
  * LoadUnboundValue: neue Methode
    Verwendet um den Wert eines unbound BOCs zu setzen, analog zu LoadValue, nur dass der Wert übergeben werden muss, 
    anstatt ihn von der DataSource zu laden.
  
- BocBooleanValue
  * Bugfix: Wurde Value auf NaBoolean gesetzt, hat das zu einer CastException geführt.

- BocCheckBox
  * Bugfix: Wurde Value auf NaBoolean gesetzt, hat das zu einer CastException geführt.
  
- BocDateTimeValue
  * Bugfix: Wurde Value auf NaDateTime gesetzt, hat das zu einer CastException geführt.
  
- BocList
  * LoadValue: Neues Verhalten bei gebundenen BocListen: 
      interim = false beendet Inline Editing und verwirft die Änderungen. 
  * SaveValue: Neues Verhalten: 
      Inline Editing wird nur beendet wenn die BocList gebunden ist.
      EndEditDetailsMode(true) ist statt dessen bei unbebundenen BocListen zu verwenden.
